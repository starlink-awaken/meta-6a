# Full Analysis - 完整分析工作流

> meta-6a skill 的完整分析模式
>
> 用途：对复杂项目进行全面的结构化分析，覆盖完整的 5A 核心框架（或 5A + 扩展模块）

## 触发条件

当用户使用以下关键词时触发：
- "完整6A" "6A分析" "deep-analysis" "完整分析" "深度分析"

## 分析过程

AI 应该对以下阶段进行完整分析（每个阶段都输出）：

### Step 1: 判断是否需要扩展模块

**AIAgent 扩展判断**：
- 本项目是否需要设计 AI 智能体？
  - **是** → 使用 5A + AIAgent 扩展流程
  - **否** → 使用标准 5A 核心流程

### Step 2: 执行完整分析

---

**1️⃣ AIAnalyze - 问题分析**
- 核心问题：[详细描述问题本质]
- 问题边界：[明确范围，什么在范围内，什么不在]
- 相关方：[列出所有利益相关者]
- 约束条件：[时间、资源、技术等约束]
- 假设：[列出所有假设，需要验证的标记出来]

**2️⃣ AIArchitect - 方案设计**
- 架构概览：[整体架构描述]
- 关键组件：[列出核心组件及其职责]
- 技术选型：[说明选择理由和替代方案]
- 接口定义：[主要接口和数据流]
- 风险识别：[技术风险和缓解措施]

**🔌 AIAgent 扩展**（如适用）
- 决策点：本项目是否需要设计 AI 智能体？
  - [是] → 角色定义：[描述智能体的角色和职责]
          → 能力规划：[列出需要的能力]
          → 交互设计：[说明与其他组件的交互方式]
  - [否] → 跳过扩展，进入 AIBuild

**3️⃣ AIBuild - 产物构建**
- 代码结构：[目录结构和模块划分]
- 数据模型：[核心数据结构和关系]
- 文档规划：[需要编写的文档列表]
- 资源需求：[人力、计算资源、第三方服务等]

**4️⃣ AIApply - 部署验证**
- 部署方案：[部署架构和流程]
- 测试策略：[单元测试、集成测试、端到端测试]
- 验收标准：[明确的成功标准]
- 监控指标：[需要监控的关键指标]

**5️⃣ AIImprove - 优化改进**
- 改进方向：[短期、中期、长期改进建议]
- 迭代计划：[分阶段的迭代路线图]
- 风险监控：[持续关注的风险点]
- 学习总结：[从这次分析中获得的经验]

---

## 质量门控

在输出前，AI 应该自检：

- [ ] 所有 5 个核心阶段都已覆盖
- [ ] 扩展模块判断有明确结论
- [ ] 每个阶段都有具体的输出内容
- [ ] 建议都是可执行的
- [ ] 关键行动项已提取

## 特殊处理

### 如果项目已经很完整

如果用户的需求或设计已经很完整，AI 应该简化相应阶段的内容，但仍需覆盖所有阶段。

### 如果缺少关键信息

如果某些关键信息缺失，AI 应该：

```
⚠️ 信息不足提醒

在 [阶段名称] 阶段发现信息缺失：
• [缺失信息1] - 影响：[描述]
• [缺失信息2] - 影响：[描述]

建议：在进入 [下一阶段] 之前，先澄清以上信息。
```

### 如果是特定领域项目

根据项目类型，某些阶段可能需要特别关注：

- **AI Agent 项目**：启用 AIAgent 扩展
- **数据工程**：强调数据模型和处理流程
- **前端项目**：强调 UI/UX 和交互设计
- **后端项目**：强调架构和接口设计

## 时间限制

- 分析时间：< 30秒
- 输出时间：< 30秒
- 总计：< 60秒

## 与快速验证的对比

| 维度 | 快速验证 | 完整分析 |
|------|----------|----------|
| 覆盖阶段 | 关键阶段 | 全部 5 个核心阶段（+扩展） |
| 输出词数 | < 200词 | 无限制，全面覆盖 |
| 时间 | < 10秒 | < 60秒 |
| 适用场景 | 日常快速检查 | 重要项目决策 |
| 决策点 | 简化处理 | 明确分析 |
