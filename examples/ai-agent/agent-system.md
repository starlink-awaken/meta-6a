# AI Agent 系统设计

**场景**：AI Agent 系统设计，需要验证架构完整性

**类型**：ai-agent

**用户输入**：
```
6A分析 '设计一个智能客服 Agent，支持意图识别、知识库检索、多轮对话'
```

**AI 输出**：
```
✅ 6A快速验证

📊 问题识别
- 核心问题：功能清晰但缺少架构设计和数据流定义
- 完整性：部分完整 - 缺少对话管理、知识库结构、兜底策略
- 紧急程度：中 - 核心功能明确，需要架构细节
- 影响范围：NLP 模块、知识库、对话管理

🎯 关键建议
1. 意图识别：使用分类模型（如 BERT）或 LLM 提取，定义意图类别体系
2. 知识库设计：向量数据库（如 Milvus/Pinecone）做语义检索 + 结构化数据做精确查询
3. 对话管理：维护对话上下文（Session 存储），设计多轮对话状态机

💡 下一步行动
绘制 Agent 架构图：Input → Intent → Router → KB-Retriever → LLM-Generator → Output

⚠️ 风险提示
知识库检索准确率是关键，建议混合检索（向量+关键词）提升召回率；LLM 幻觉问题需要事实核查机制。
```

**价值**：
- 展示 AI Agent 系统设计的验证方法
- 适用于智能客服、RAG 系统、Agent 应用设计
- 强调意图识别、知识库设计、对话管理的关键要点
