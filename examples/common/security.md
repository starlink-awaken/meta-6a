# 安全审查

**场景**：代码安全审查，识别潜在漏洞

**类型**：common

**用户输入**：
```
6A检查 "用户登录接口直接把密码拼接到 SQL 查询中"
```

**AI 输出**：
```
✅ 6A快速验证

📊 问题识别
- 核心问题：严重安全漏洞 - SQL 注入风险
- 完整性：完整 - 问题明确
- 紧急程度：🔴 极高 - 可导致数据泄露
- 影响范围：所有用户数据

🎯 关键建议
1. 立即修复：使用参数化查询（prepared statements）
2. 添加输入验证：限制密码长度和字符集
3. 审计其他接口：检查是否存在类似问题

💡 下一步行动
• 短期：立即修复 SQL 注入漏洞
• 中期：进行全面安全审计
• 长期：建立安全代码审查流程

🚨 严重风险
这是 SQL 注入漏洞，攻击者可以：
- 窃取所有用户密码
- 绕过认证登录任意账户
- 删除整个数据库
必须立即修复！
```

**价值**：
- 展示严重安全漏洞的识别和响应流程
- 适用于安全审查和漏洞修复
- 强调立即修复和建立长期安全流程的重要性
