# 项目规划

**场景**：用户要做项目，需要结构化分析

**类型**：common

**用户输入**：
```
对这个项目做快速验证："开发一个博客系统，支持文章发布、评论、标签"
```

**AI 输出**：
```
✅ 6A快速验证

📊 问题识别
- 核心问题：功能清晰但缺少优先级和分阶段规划
- 完整性：部分完整 - 缺少MVP定义、技术栈、时间规划
- 紧急程度：中 - 功能明确，需要规划
- 影响范围：内容管理、评论系统、标签管理

🎯 关键建议
1. 定义 MVP：第一版只做文章发布和列表，评论和标签后续迭代
2. 技术栈选择：考虑使用 CMS（如 Strapi、Ghost）而非从零开发
3. 分阶段计划：
   - Phase 1: 文章CRUD（2周）
   - Phase 2: 评论系统（1周）
   - Phase 3: 标签系统（1周）

💡 下一步行动
进入 AIArchitect 阶段，绘制功能架构图和分阶段计划。

⚠️ 风险提示
从零开发 CMS 工作量大，建议评估是否使用现有方案。
```

**价值**：
- 展示项目规划的 MVP 思维
- 适用于新项目的结构化分析和分阶段规划
- 提供技术选型和优先级排序的参考
