# 设计验证

**场景**：用户有技术方案，需要验证可行性

**类型**：common

**用户输入**：
```
检查这个设计："用 Redis 做缓存，MySQL 做主存储，前端用 Vue3"
```

**AI 输出**：
```
✅ 6A快速验证

📊 问题识别
- 核心问题：技术栈合理，但缺少架构细节
- 完整性：部分完整 - 缺少缓存策略、数据同步、容错机制
- 紧急程度：低 - 技术选型合理
- 影响范围：缓存层、数据库、前端框架

🎯 关键建议
1. 定义缓存策略：哪些数据缓存？TTL设置？缓存穿透/雪崩如何处理？
2. 数据同步机制：MySQL → Redis 的更新策略
3. 容错设计：Redis 宕机如何降级？数据一致性如何保证？

💡 下一步行动
进入 AIArchitect 阶段，绘制系统架构图和数据流图。

⚠️ 风险提示
缓存和数据库的双写一致性是常见难点，需要仔细设计。
```

**价值**：
- 展示技术选型验证的方法论
- 适用于架构设计初期的快速审查
- 提供缓存、数据库等关键组件的验证要点
