# Backend API 设计

**场景**：后端 API 接口设计，需要验证规范性和完整性

**类型**：backend

**用户输入**：
```
检查6A '设计一个用户管理 REST API，包括 CRUD 操作、分页、搜索'
```

**AI 输出**：
```
✅ 6A快速验证

📊 问题识别
- 核心问题：API 基础功能明确，但缺少规范细节
- 完整性：部分完整 - 缺少接口规范、错误码、认证鉴权
- 紧急程度：中 - 可开始设计，需补充规范
- 影响范围：用户服务、API 网关、前端对接

🎯 关键建议
1. 接口规范：遵循 RESTful 风格，GET/POST/PUT/DELETE 语义明确
2. 分页设计：推荐 cursor-based 或 offset-based？返回 total 字段
3. 搜索实现：全文搜索用 Elasticsearch？简单筛选用数据库 LIKE

💡 下一步行动
定义 API 文档（Swagger/OpenAPI），明确请求/响应结构、状态码、错误格式。

⚠️ 风险提示
缺少认证鉴权设计可能导致未授权访问；搜索功能需注意 SQL 注入风险。
```

**价值**：
- 展示 REST API 设计的验证维度
- 适用于后端接口设计和规范制定
- 强调接口规范、分页设计、搜索实现的关键决策点
