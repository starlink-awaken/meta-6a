# Frontend 组件设计

**场景**：React 组件设计，需要验证组件架构和实现方案

**类型**：frontend

**用户输入**：
```
验证一下 '设计一个用户列表 React 组件，支持分页、筛选、排序'
```

**AI 输出**：
```
✅ 6A快速验证

📊 问题识别
- 核心问题：组件功能清晰，但缺少状态管理和数据流设计
- 完整性：部分完整 - 缺少组件拆分、API 集成、加载状态
- 紧急程度：中 - 需要明确组件架构
- 影响范围：列表页、API 层、状态管理

🎯 关键建议
1. 组件拆分：UserList（容器）、UserTable（展示）、Pagination（分页）、FilterBar（筛选）
2. 状态管理：使用 URL 参数同步筛选/排序/分页状态，支持刷新和分享
3. 性能优化：使用 React.memo 虚拟列表渲染大量数据时使用 react-window

💡 下一步行动
使用 TanStack Query 管理 API 状态，Zustand/Jotai 管理筛选条件，实现组件分层。

⚠️ 风险提示
避免在 useEffect 中直接修改状态作为依赖项，可能导致无限循环。
```

**价值**：
- 展示前端组件设计的架构思维
- 适用于 React/Vue 等现代前端框架的组件设计
- 强调组件拆分、状态管理、性能优化的最佳实践
